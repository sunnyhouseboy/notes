    在本实验中，你将：

    • 利用在前一个实验中开发的多变量程序

    多变量梯度下降

    以下是你上一个实验中开发的多变量梯度下降方程：

    $$\begin{align*} \text{repeat}&\text{until convergence：} ; \lbrace \newline;

    & w_j := w_j - \alpha \frac{\partial J(\mathbf{w},b)}{\partial w_j} \tag{1} ; & \text{for j = 0..n-1}\newline

    &b\ \ := b - \alpha \frac{\partial J(\mathbf{w},b)}{\partial b} \newline \rbrace

    \end{align*}$$

    其中，$n$ 是特征数量，参数 $w_j$ 和 $b$ 同时更新，并且：

    $$\begin{align} \frac{\partial J(\mathbf{w},b)}{\partial w_j} &= \frac{1}{m} \sum\limits_{i = 0}^{m-1} (f_{\mathbf{w},b}(\mathbf{x}^{(i)}) - y^{(i)})x_{j}^{(i)} \tag{2} \ \frac{\partial J(\mathbf{w},b)}{\partial b} &= \frac{1}{m} \sum\limits_{i = 0}^{m-1} (f_{\mathbf{w},b}(\mathbf{x}^{(i)}) - y^{(i)}) \tag{3} \end{align}$$

    • $m$ 是数据集中的训练样本数量

    • $f_{\mathbf{w},b}(\mathbf{x}^{(i)})$ 是模型的预测值，而 $y^{(i)}$ 是目标值

    特征缩放

    ![[C1_W2_Lab06_featurescalingheader.png]] 课程中介绍了重缩放数据集（rescaling the dataset）的重要性，以确保各特征具有相似的取值范围。

    原理

    让我们再次回顾 $\alpha$ = 9e-7 时的情况。这已经非常接近我们在不发散的前提下所能设置的 $\alpha$ 最大值了。这是一个显示前几次迭代的短期运行结果： ![[C1_W2_Lab06_ShortRun.png]] 如上所示，虽然成本正在降低，但显而易见的是，由于 $w_0$ 的梯度大得多，它的进展比其他参数要快得多。

    下图显示了在 $\alpha$ = 9e-7 条件下进行超长运行的结果。这需要花费数小时。 ![[C1_W2_Lab06_LongRun.png]] 在上方你可以看到，成本在初始下降后减速非常缓慢。请注意 w0 与 w1、w2、w3 之间的差异，以及 dj_dw0 与 dj_dw1-3 之间的区别。w0 很快就达到了接近最终值的状态，且 dj_dw0 迅速减小到一个很小的值，表明 w0 已接近最终解。而其他参数的下降速度则慢得多。

    这是为什么呢？有什么我们可以改进的地方吗？请看下图： ![[C1_W2_Lab06_scale.png]] 上图展示了为什么 $w$ 的更新是不均匀的：

    • 所有的参数更新（$w$ 和 $b$）都共用同一个 $\alpha$。

    • 在更新 $w$ 时，共同的误差项会乘以对应的特征（$b$ 则不用）。

    • 特征的大小差异显著，导致某些特征的更新速度比其他特征快得多。在这种情况下，$w_0$ 乘以的是“尺寸（平方英尺）”，该数值通常 > 1000；而 $w_1$ 乘以的是“卧室数量”，通常在 2-4 之间。

    解决方案就是特征缩放（Feature Scaling）。

    课程中讨论了三种不同的技术：

    • 特征缩放（Feature scaling）：本质上是将每个正特征除以其最大值；或者更通用地，利用 $(x-min)/(max-min)$ 根据最小值和最大值对每个特征进行重缩放。这两种方法都能将特征归一化到 -1 到 1 的范围。前者适用于正特征，操作简单且适用于课程中的示例；后者则适用于任何特征。

    • 均值归一化（Mean normalization）：$x_i := \frac{x_i - \mu_i}{max - min}$

    • Z-score 归一化（Z-score normalization）：我们将在下文中探讨这种方法。

    Z-score 标准化

    Z-score 标准化的计算公式如下：

    $$x_j^{(i)} = \frac{x_j^{(i)} - \mu_j}{\sigma_j} \tag{4}$$

    其中 $j$ 是特征索引。$\mu_j$ 是特征 $j$ 的平均值，$\sigma_j$ 是其标准差。

    $$\begin{align}

    \mu_j &= \frac{1}{m} \sum_{i=0}^{m-1} x_j^{(i)} \tag{5} \

    \sigma^2_j &= \frac{1}{m} \sum_{i=0}^{m-1} (x_j^{(i)} - \mu_j)^2 \tag{6}

    \end{align}$$

    注意： 在对训练集进行标准化时，必须存储用于计算的平均值和标准差。在应用模型进行预测时，必须使用这些存储的值对输入数据进行同样的缩放。

    def zscore_normalize_features(X):
    """
    计算 X 的 z-score 标准化。

    参数:
      X (ndarray (m,n))     : 输入数据，m 个样本，n 个特征

    返回:
      X_norm (ndarray (m,n)): 归一化后的输入
      mu (ndarray (n,))     : 每个特征的平均值
      sigma (ndarray (n,))  : 每个特征的标准差
    """
    # 找到每个特征（列）的平均值
    mu     = np.mean(X, axis=0)                 # mu 的形状为 (n,)
    # 找到每个特征（列）的标准差
    sigma  = np.std(X, axis=0)                  # sigma 的形状为 (n,)
    # 逐元素相减，并按列除以 sigma
    X_norm = (X - mu) / sigma                     

    return (X_norm, mu, sigma)

# 检查操作：可以使用工业级的机器学习库 sklearn来实现和手写的 `zscore_normalize_features` 函数完全相同的功能。

    • 生成图表时，使用的是归一化后的特征。任何使用从归一化训练集中学到的参数进行的预测，其输入的特征也必须经过归一化处理。

    预测：构建模型的目的在于利用它来预测不在数据集中的房屋价格。让我们预测一间拥有 1200 平方英尺、3 间卧室、1 层楼、房龄 40 年的房屋价格。请记住，你必须使用在训练数据归一化时得到的均值（mean）和标准差（standard deviation）来对这些数据进行归一化。

    # 首先，标准化我们的预测目标
x_house = np.array([1200, 3, 1, 40])
x_house_norm = (x_house - X_mu) / X_sigma
print(x_house_norm)
x_house_predict = np.dot(x_house_norm, w_norm) + b_norm
print(f"预测 1200 平方英尺, 3 间卧室, 1 层, 40 年房龄的房价 = ${x_house_predict*1000:0.0f}")
