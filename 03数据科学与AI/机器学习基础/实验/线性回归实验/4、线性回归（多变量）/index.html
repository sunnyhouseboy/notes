    多变量线性回归 (Multiple Variable Linear Regression)

    1 概述

    1.1 目标

    • 扩展回归模型例程以支持多个特征

      • 扩展数据结构以支持多个特征

      • 重写预测、代价和梯度例程以支持多变量

      • 利用 NumPy 的 np.dot 进行向量化实现，以提高速度并简化代码

    1.2 工具

    在本实验中，我们将使用：

    • NumPy：一个流行的科学计算库

    • Matplotlib：一个流行的绘图库

    import copy, math
import numpy as np
import matplotlib.pyplot as plt
plt.style.use('./deeplearning.mplstyle')

    1.3 记法

    以下是针对多特征更新后的记法摘要：

    通用记法 描述 Python (如果适用)
    $a$ 标量，非加粗
    $\mathbf{a}$ 向量，加粗小写
    $\mathbf{A}$ 矩阵，加粗大写
    回归
    $\mathbf{X}$ 训练样本矩阵 X_train
    $\mathbf{y}$ 训练样本目标值 y_train
    $\mathbf{x}^{(i)}$, $y^{(i)}$ 第 $i$ 个训练样本 X[i], y[i]
    $m$ 训练样本数量 m
    $n$ 每个样本中的特征数量 n
    $\mathbf{w}$ 参数：权重 (weight) w
    $b$ 参数：偏置 (bias) b
    $f_{\mathbf{w},b}(\mathbf{x}^{(i)})$ 在由 $\mathbf{w},b$ 参数化的 $\mathbf{x}^{(i)}$ 处的模型评估结果：$f_{\mathbf{w},b}(\mathbf{x}^{(i)}) = \mathbf{w} \cdot \mathbf{x}^{(i)}+b$ f_wb

    2 问题陈述

    你将使用房价预测的案例。训练数据集包含三个样本，具有四个特征（面积、卧室数量、层数和房龄），如下表所示。请注意，与之前的实验不同，面积单位是平方英尺 (sqft) 而非 1000 平方英尺。这会导致一个问题，你将在下一个实验中解决它！

    面积 (sqft) 卧室数量 层数 房龄 价格 (单位: 1000美元)
    2104 5 1 45 460
    1416 3 2 40 232
    852 2 1 35 178

    你将使用这些值构建一个线性回归模型，以便预测其他房子的价格。例如，一个 1200 sqft、3 个卧室、1 层、40 年房龄的房子。

    请运行以下代码单元来创建你的 X_train 和 y_train 变量。

    X_train = np.array([[2104, 5, 1, 45], [1416, 3, 2, 40], [852, 2, 1, 35]])
y_train = np.array([460, 232, 178])

    2.1 包含样本的矩阵 X

    类似于上表，样本存储在 numpy 矩阵 X_train 中。矩阵的每一行代表一个样本。当你拥有 $m$ 个训练样本（在我们的例子中 $m$ 为 3），且有 $n$ 个特征（在我们的例子中 $n$ 为 4）时，$\mathbf{X}$ 是一个维度为 ($m$, $n$) 的矩阵（m 行，n 列）。

    $$\mathbf{X} = \begin{pmatrix} x^{(0)}0 & x^{(0)}_1 & \cdots & x^{(0)}{n-1} \ x^{(1)}0 & x^{(1)}_1 & \cdots & x^{(1)}{n-1} \ \cdots \ x^{(m-1)}0 & x^{(m-1)}_1 & \cdots & x^{(m-1)}{n-1} \end{pmatrix}$$

    记法 (notation)：

    • $\mathbf{x}^{(i)}$ 是包含样本 $i$ 的向量。$\mathbf{x}^{(i)} = (x^{(i)}0, x^{(i)}_1, \cdots, x^{(i)}{n-1})$

    • $x^{(i)}_j$ 是样本 $i$ 中的元素 $j$。括号中的上标表示样本编号，而下标表示元素。

    显示输入数据：

    # 数据存储在 numpy 数组/矩阵中
print(f"X Shape: {X_train.shape}, X Type:{type(X_train)})")
print(X_train)
print(f"y Shape: {y_train.shape}, y Type:{type(y_train)})")
print(y_train)

    输出：

    X Shape: (3, 4), X Type:) [[2104 5 1 45] [1416 3 2 40] [ 852 2 1 35]] y Shape: (3,), y Type:) [460 232 178]

    2.2 参数向量 w, b

    • $\mathbf{w}$ 是一个具有 $n$ 个元素的向量。
    • 每个元素包含一个与特征相关的参数。
    • 在我们的数据集中，$n=4$。
    • 在记法上，我们将其绘制为列向量。

    $$\mathbf{w} = \begin{pmatrix} w_0 \ w_1 \ \cdots\ w_{n-1} \end{pmatrix}$$ - $b$ 是一个标量参数。

    为了演示，我们将为 $\mathbf{w}$ 和 $b$ 加载一些接近最优值的初始值。$\mathbf{w}$ 是一个 1-D NumPy 向量。

    b_init = 785.1811367994083
w_init = np.array([ 0.39133535, 18.75376741, -53.36032453, -26.42131618])
print(f"w_init shape: {w_init.shape}, b_init type: {type(b_init)}")

    3 多变量模型预测

    多变量模型的预测由线性模型给出：

    $$ f_{\mathbf{w},b}(\mathbf{x}) = w_0x_0 + w_1x_1 +... + w_{n-1}x_{n-1} + b \tag{1}$$

    或使用向量记法：

    $$f_{\mathbf{w},b}(\mathbf{x}) = \mathbf{w} \cdot \mathbf{x} + b \tag{2}$$

    其中 $\cdot$ 是向量的“点积 (dot product)”。

    为了演示点积，我们将使用 (1) 和 (2) 来实现预测。

    3.1 逐元素进行单次预测

    我们之前的预测是将一个特征值乘以一个参数并加上偏置参数。将之前的预测实现直接扩展到多特征，就是使用循环遍历每个元素，执行与对应参数的乘法，最后在结尾加上偏置。

    def predict_single_loop(x, w, b): 
    """
    使用线性回归进行单次预测

    参数:
      x (ndarray): 形状为 (n,)，具有多个特征的样本
      w (ndarray): 形状为 (n,)，模型参数    
      b (标量):    模型参数     

    返回:
      p (标量):    预测值
    """
    n = x.shape[0]
    p = 0
    for i in range(n):
        p_i = x[i] * w[i]  
        p = p + p_i         
    p = p + b                
    return p

    # 从训练数据中获取一行
x_vec = X_train[0,:]
print(f"x_vec shape {x_vec.shape}, x_vec value: {x_vec}")

# 进行预测
f_wb = predict_single_loop(x_vec, w_init, b_init)
print(f"f_wb shape {f_wb.shape}, prediction: {f_wb}")

    输出：

    x_vec shape (4,), x_vec value: [2104 5 1 45]

    f_wb shape (), prediction: 459.9999976194083

    请注意 x_vec 的形状。它是一个具有 4 个元素的 1-D NumPy 向量 (4,)。结果 f_wb 是一个标量。

    3.2 向量化单次预测

    注意到上面的公式 (1) 可以使用公式 (2) 中的点积来实现。我们可以利用向量运算来加速预测。

    回想一下之前的 Python/NumPy 实验，NumPy 的 np.dot() 可以用来执行向量点积。

    def predict(x, w, b): 
    """
    使用线性回归进行单次预测
    参数:
      x (ndarray): 形状为 (n,)，具有多个特征的样本
      w (ndarray): 形状为 (n,)，模型参数   
      b (标量):             模型参数 

    返回:
      p (标量):    预测值
    """
    p = np.dot(x, w) + b     
    return p    

    # 从训练数据中获取一行
x_vec = X_train[0,:]
print(f"x_vec shape {x_vec.shape}, x_vec value: {x_vec}")

# 进行预测
f_wb = predict(x_vec, w_init, b_init)
print(f"f_wb shape {f_wb.shape}, prediction: {f_wb}")

    输出：

    x_vec shape (4,), x_vec value: [2104 5 1 45]

    f_wb shape (), prediction: 459.99999761940825

    结果和形状与使用循环的上一个版本相同。往后，我们将使用 np.dot 进行此类操作。预测现在变成了一条简单的语句。大多数例程将直接实现它，而不是调用单独的预测例程。

    4 计算多变量代价

    多变量代价函数 $J(\mathbf{w},b)$ 的等式为：

    $$J(\mathbf{w},b) = \frac{1}{2m} \sum\limits_{i = 0}^{m-1} (f_{\mathbf{w},b}(\mathbf{x}^{(i)}) - y^{(i)})^2 \tag{3}$$

    其中：

    $$f_{\mathbf{w},b}(\mathbf{x}^{(i)}) = \mathbf{w} \cdot \mathbf{x}^{(i)} + b \tag{4}$$

    与之前的实验不同，$\mathbf{w}$ 和 $\mathbf{x}^{(i)}$ 现在是支持多特征的向量而非标量。

    下面是公式 (3) 和 (4) 的实现。请注意，这使用了本课程的_标准模式_，即对所有 $m$ 个样本使用 for 循环。

    def compute_cost(X, y, w, b): 
    """
    计算代价
    参数:
      X (ndarray (m,n)): 数据，m 个样本，n 个特征
      y (ndarray (m,)) : 目标值
      w (ndarray (n,)) : 模型参数  
      b (标量)       : 模型参数

    返回:
      cost (标量): 代价
    """
    m = X.shape[0]
    cost = 0.0
    for i in range(m):                                
        f_wb_i = np.dot(X[i], w) + b           # (n,)(n,) = 标量
        cost = cost + (f_wb_i - y[i])**2       # 标量
    cost = cost / (2 * m)                      # 标量    
    return cost

    # 使用我们预先选定的最优参数计算并显示代价。
cost = compute_cost(X_train, y_train, w_init, b_init)
print(f'Cost at optimal w : {cost}')

    预期结果：

    Cost at optimal w : 1.5578904045996674e-12

    5 多变量梯度下降

    多变量梯度下降：

    $$\begin{align} \text{重复直至收敛:} \; \lbrace \ & w_j = w_j - \alpha \frac{\partial J(\mathbf{w},b)}{\partial w_j} \tag{5} \; \text{对于 j = 0..n-1}\ &b = b - \alpha \frac{\partial J(\mathbf{w},b)}{\partial b} \ \rbrace \end{align}$$

    其中，$n$ 是特征数量，参数 $w_j$ 和 $b$ 同时更新，且

    $$\begin{align} \frac{\partial J(\mathbf{w},b)}{\partial w_j} &= \frac{1}{m} \sum\limits_{i = 0}^{m-1} (f_{\mathbf{w},b}(\mathbf{x}^{(i)}) - y^{(i)})x_{j}^{(i)} \tag{6} \ \frac{\partial J(\mathbf{w},b)}{\partial b} &= \frac{1}{m} \sum\limits_{i = 0}^{m-1} (f_{\mathbf{w},b}(\mathbf{x}^{(i)}) - y^{(i)}) \tag{7} \end{align}$$

    • $m$ 是数据集中的训练样本数

    • $f_{\mathbf{w},b}(\mathbf{x}^{(i)})$ 是模型的预测值，而 $y^{(i)}$ 是目标值

    5.1 计算多变量梯度

    计算等式 (6) 和 (7) 的实现如下。实现方式有很多种。在这个版本中，包含：

    • 一个遍历所有 $m$ 个样本的外层循环。

      • 样本的 $\frac{\partial J(\mathbf{w},b)}{\partial b}$ 可以直接计算并累加。

      • 在第二个遍历所有 $n$ 个特征的循环中：

        • 为每个 $w_j$ 计算 $\frac{\partial J(\mathbf{w},b)}{\partial w_j}$。
    def compute_gradient(X, y, w, b): 
    """
    计算线性回归的梯度 
    参数:
      X (ndarray (m,n)): 数据，m 个样本，n 个特征
      y (ndarray (m,)) : 目标值
      w (ndarray (n,)) : 模型参数  
      b (标量)       : 模型参数

    返回:
      dj_dw (ndarray (n,)): 代价相对于参数 w 的梯度。 
      dj_db (标量):       代价相对于参数 b 的梯度。 
    """
    m,n = X.shape           # (样本数, 特征数)
    dj_dw = np.zeros((n,))
    dj_db = 0.

    for i in range(m):                             
        err = (np.dot(X[i], w) + b) - y[i]   
        for j in range(n):                         
            dj_dw[j] = dj_dw[j] + err * X[i, j]    
        dj_db = dj_db + err                        
    dj_dw = dj_dw / m                                
    dj_db = dj_db / m                                

    return dj_db, dj_dw

    # 计算并显示梯度 
tmp_dj_db, tmp_dj_dw = compute_gradient(X_train, y_train, w_init, b_init)
print(f'dj_db at initial w,b: {tmp_dj_db}')
print(f'dj_dw at initial w,b: \n {tmp_dj_dw}')

    结果：

    dj_db at initial w,b: -1.6739251122999121e-06
    dj_dw at initial w,b:
    [-2.73e-03 -6.27e-06 -2.22e-06 -6.92e-05]

    5.2 多变量梯度下降迭代

    下面的例程实现了上述等式 (5)。

    def gradient_descent(X, y, w_in, b_in, cost_function, gradient_function, alpha, num_iters): 
    """
    执行批梯度下降以学习权重。通过执行 num_iters 步梯度步骤，以学习率 alpha 更新权重

    参数:
      X (ndarray (m,n))   : 数据，m 个样本，n 个特征
      y (ndarray (m,))    : 目标值
      w_in (ndarray (n,)) : 初始模型参数  
      b_in (标量)       : 初始模型参数
      cost_function       : 计算代价的函数
      gradient_function   : 计算梯度的函数
      alpha (float)       : 学习率
      num_iters (int)     : 运行梯度下降的迭代次数

    返回:
      w (ndarray (n,)) : 更新后的参数值 
      b (标量)       : 更新后的参数值 
      """

    # 存储代价 J 和 w 的数组，用于后续绘图
    J_history = []
    w = copy.deepcopy(w_in)  # 避免在函数内修改全局变量 w
    b = b_in

    for i in range(num_iters):

        # 计算梯度并更新参数
        dj_db,dj_dw = gradient_function(X, y, w, b)

        # 使用 w, b, alpha 和梯度更新参数
        w = w - alpha * dj_dw
        b = b - alpha * dj_db

        # 保存每次迭代的代价 J
        if i<100000:      # 防止资源枯竭 
            J_history.append( cost_function(X, y, w, b))

        # 每隔 1/10 迭代次数打印一次代价
        if i% math.ceil(num_iters / 10) == 0:
            print(f"Iteration {i:4d}: Cost {J_history[-1]:8.2f}   ")

    return w, b, J_history # 返回最终的 w, b 和 J 历史记录用于绘图

    在下一个单元格中，你将测试该实现。

    # 初始化参数
initial_w = np.zeros_like(w_init)
initial_b = 0.
# 梯度下降设置
iterations = 1000
alpha = 5.0e-7
# 运行梯度下降 
w_final, b_final, J_hist = gradient_descent(X_train, y_train, initial_w, initial_b,
compute_cost, compute_gradient, alpha, iterations)
print(f"b,w found by gradient descent: {b_final:0.2f},{w_final} ")
m,_ = X_train.shape
for i in range(m):
    print(f"prediction: {np.dot(X_train[i], w_final) + b_final:0.2f}, target value: {y_train[i]}")

ax2.plot(100 + np.arange(len(J_hist[100:])), J_hist[100:])

ax1.set_ylabel('Cost')             ;  ax2.set_ylabel('Cost') 
ax1.set_xlabel('iteration step')   ;  ax2.set_xlabel('iteration step')

    6 结论

    6.1：一次“令人失望”的成功

    • 成功之处： 我们成功地将单变量线性回归升级成了多变量模型，并验证了逻辑的正确性。

    • 失望之处： 预测出来的房价和真实房价差距依然很大，模型并没有真正收敛。

    6.2向量化（Vectorization）是工程落地的必经之路

    从代码实现的角度来看，这个实验最重要的技术结论是：在多变量（多特征）甚至海量数据的场景下，绝对不能用纯 Python 的 for 循环去挨个计算特征和权重的乘积。 必须使用线性代数中的点积（Dot Product），也就是实验中重点使用的 np.dot()。这是摆脱低效计算、利用底层 C/Fortran 库加速的关键，也是以后理解现代深度学习框架（如 PyTorch, TensorFlow）张量运算的底层基础。

    6.3 数据质量与算法同等重要（引出特征缩放）

    这也是这个实验特意留下的一个“教学悬念”。它用一个失败的预测结果告诉我们一个硬道理：再优雅的代码（向量化），如果直接喂给它未经处理的、量级悬殊的原始数据（比如面积2104 vs 楼层1），梯度下降算法也会被“拖垮”。

    最终的推论与下一步预告： 为了让梯度下降在多变量场景下高效工作，除了优化代码逻辑，必须对数据进行预处理。这就是作者在结尾提到的—— 下一个实验必然会引入特征缩放（Feature Scaling）技术（比如让所有特征的数据都缩放到 -1 到 1 之间），从而彻底解决这个问题。
