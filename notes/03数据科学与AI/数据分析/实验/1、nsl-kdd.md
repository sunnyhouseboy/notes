# 二、Label 攻击类型分类及特征对照表
| **攻击大类 (Category)** | **包含的细分标签 (部分常见)**                              | **攻击逻辑与特征表现**                                                  | **重要程度** |
| ------------------- | ----------------------------------------------- | -------------------------------------------------------------- | -------- |
| **Normal**          | `normal`                                        | 正常的网络行为。标志位多为 SF，`serror_rate` 接近 0。                           | **★ 核心** |
| **DoS** (拒绝服务)      | ==`neptune`==, `smurf`, `back`, `teardrop`      | 旨在瘫痪目标。**特征：** `count` 极大，`serror_rate` 极高，或者 `src_bytes` 异常大。 | **★ 核心** |
| **Probe** (扫描)      | ==`ipsweep`==, ==`satan`==, `nmap`, `portsweep` | 刺探目标信息。**特征：** `dst_host_count` 较高，但单个连接的 `duration` 很短。       | **★ 核心** |
| **R2L** (远程访问)      | `guess_passwd`, `warezmaster`, `ftp_write`      | 尝试非法进入。**特征：** `num_failed_logins` 增加，`hot` 计数高。               | **★ 重要** |
| **U2R** (提权)        | `buffer_overflow`, `rootkit`, `loadmodule`      | 已经是用户，尝试拿 root。**特征：** 流量极小但很危险，`root_shell` 为 1。              | **★ 重要** |
### 1、 DoS（拒绝服务：旨在瘫痪目标）

这类攻击的核心目的就是“搞破坏”，让服务器瘫痪、资源耗尽。

- **`neptune`**：即经典的 SYN Flood 攻击。攻击者疯狂向服务器发送“建立连接”的请求但不完成最后一步，导致服务器的“接待处”被占满，正常用户挤不进去。
    
- **`smurf`**：利用广播地址的“借刀杀人”式泛洪攻击。伪造受害者的IP向全网发送Ping请求，让海量无辜主机的应答包把受害者彻底淹没（这正是咱们刚才聊过的“恶意点外卖”）。
    
- **`back`**：针对 Apache Web 服务器的攻击。在网址（URL）里塞满超长、畸形的请求字符（比如无数个 `/`），让服务器在处理这些垃圾请求时卡死宕机。
    
- **`teardrop`**：发出一堆互相重叠、碎片化的错误数据包。目标机器在尝试把这些碎片“拼图”拼装还原时会发生严重混乱，最终导致系统崩溃。
    

---

### 2、 Probe（扫描探测：踩点收集信息）

这类攻击不直接破坏，而是像“小偷踩点”，寻找可乘之机。

- **`ipsweep`**：IP 地址扫街。挨个发送探测包问“有人在吗？”，目的是快速摸清目标网络里有哪些主机是开机的、活跃的。
    
- **`portsweep`**：端口扫街。针对一台或多台主机，挨个试探它的哪些门（比如 80网页端口, 22远程登录端口）是敞开的。
    
- **`nmap`**：网络探测界的“瑞士军刀”。使用专业的扫描工具，精细化地刺探目标主机的操作系统版本、服务类型等详细的“户口本”信息。
    
- **`satan`**：自动化漏洞扫描。不仅探测主机和端口，还会顺带测试这些服务是否存在已知的安全漏洞，直接为后续入侵打基础。
    

---

### 3、 R2L（远程访问本地：尝试非法进入）

攻击者在外部网络，想方设法要拿到目标主机内部的普通访问权限。

- **`guess_passwd`**：暴力破解/字典攻击。就是躲在门外，拿着一本密码本不断尝试登录（比如 SSH 或 Telnet），特征是会出现大量的“登录失败”记录。
    
- **`ftp_write`**：利用 FTP（文件传输）服务的配置漏洞。攻击者以匿名身份，偷偷往原本不该有写入权限的服务器目录里上传恶意文件。
    
- **`warezmaster`**：盗版软件集散地。攻击者非法侵入主机（通常是 FTP 服务器）后，把它当做中转站，用来上传、存储和发布盗版软件。
    

---

### 4、 U2R（本地提权：尝试拿最高控制权）

攻击者已经混进系统成了普通用户，现在想尽办法要拿到超级管理员（Root）权限。

- **`buffer_overflow`**：缓冲区溢出。给有漏洞的程序喂超量的垃圾数据把它“撑爆”，趁系统内存混乱时，把后门代码挤进去执行，从而骗取更高权限。
    
- **`loadmodule`**：利用系统动态加载机制的漏洞。普通用户强行让系统加载一个自己伪造的补丁或驱动模块，借此摇身一变成为系统管理员。
    
- **`rootkit`**：顶级隐身衣+万能钥匙。这通常是黑客拿到高级权限后植入的底层恶意软件，用来掩盖自己的入侵踪迹，并长期死死咬住对系统的最高控制权。
